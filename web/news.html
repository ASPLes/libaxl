    Next major stable release. Base library (libaxl) was updated to allow configuring a set of external handler that allows to perform run-time buffered decode operations. This was used to implement libaxl-babel, the foundation to give Axl Library support for more encoding formats than utf-8.

  • 2006-11-09 LibAxl 0.3.0 is ready, how fast is your XML toolkit?

    This new release makes Axl Library to reach one of its objectives: to produce a high quality xml toolkit, with a clean interface, that provides good performance results at runtime execution and memory consumption.

    Using LibXML as reference, which maybe is one of the fastest xml toolkit available, this Axl release shows to be faster than it while processing xml documents and, at the same time, consuming around 2.5 times less. See the following report to get more details: Axl Library 0.3.0 performance and memory report.

    You can get full details at the release note: Axl Library 0.3.0
