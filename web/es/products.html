Productos y aplicaciones utilizando Axl Library

A continuación se muestra una lista de aplicaciones conocidas por utilizar Axl Library junto con una breve descripción sobre ellas. Contacte con nosotros a través de la lista de correo para notificar más aplicaciones utilizando Axl Library.

TML/Sidex: un constructor de perfiles BEEP

TML/SIDEX es una aproximación pragmática para resolver los problemas de desarrollo del día a día. Es. Es escalable desde pocas llamadas con unos pocos parámetros a datos e interfaces complejas con muchos comandos. Está probado en projectos de la vida real y está ejecutando todos los días en cientos de instalaciones de todo el mundo. Puede comprobar el listado de referencia y la descripción del software en http://www.wobe-team.com.

TML/SIDEX proporciona una solución integrada y lista para usar para el desarrollo de comunicaciones de red independiente de plataforma. Esta solución proporciona soporte para hacer llamadas peer to peer de manera síncrona y asíncrona.

La combinación TML/SIDEX es una poderosa solución que permite al programador implementar interfaces de comunicación rápida y fácilmente. Vea más en la siguiente página: https://www.tml-software.com/products/tml/

Vea también la siguiente introducción acerca sobre TML/SIDEX: http://prezi.com/qg0kpllkzn3l/beep-en/

  • Homepage: http://www.tml-software.com

Core-Admin: plataforma de administración de servidores & IoT

Core-Admin is una solución extensible para la administración de servidores y dispositivos que integra en una única solución paneles de administración, un motor de comprobación externo e interno detallado y un sistema de notificación que le permitirá realizar el seguimiento al minuto del estado de todos los sistemas conectados a una consola central.

Core-Admin utiliza LibAxl para soportar todos los requisitos XML que son los ficheros de configuración y algunos métodos de invocación

  • Página oficial: http://www.core-admin.com

Vortex Library: Una implementación BEEP (RFC3080/RFC3081)

Vortex Library es una implementación del RFC 3080 / RFC 3081, conocido como el protocolo "BEEP Core". Vortex Library functiona sobre la pila TCP/IP y está escrita utilizando el lenguaje C. De manera adicional, la librería incluye un soporte completo para XML-RPC sobre BEEP (RFC 3529).

  • Página oficial: http://www.aspl.es/vortex

Turbulence: servidor de aplicaciones BEEP

Turbulence es un servidor BEEP de propósito general, construido sobre Vortex Library, que proporciona muchas características administrativas por defecto.

Turbulence es extensible a través de módulos, permitiendo la implementación de perfiles BEEP en el lado servidor que luego son usados y combinados con otros perfiles a través de una configuración de ejecucción llamada: "profile path".

Turbulence está escrito con el objetivo de hacer sencillo desarrollar y desplegar perfiles BEEP, permitiendo a los desarrolladores proporcionar una interfaz de configuración a los administradores y usuarios de dichos perfiles.

Comercialmente soportada, Turbulence está liberado como un projecto de código abierto, bajo la licencia LGPL 2.1 (ver licenciamiento de Turbulence), permitiendo desarrollar productos de código abierto y comerciales.

Turbulence está desarrollado y mantenido por ASPL, y licenciado bajo los terminos de la LGPL 2.1 con la intención de promover la adopción de BEEP.

  • Página oficial: http://www.aspl.es/turbulence

MyQtt: stack MQTT para escribir servidores MQTT

MyQtt es un stack Open Source profesional del protocolo MQTT, escrito en ANSI C, que soporta las versiones v3.1.1 y v3.1, especializado en el soporte para crear servidores MQTT. MyQtt tiene un diseño modular que permite crear servidores MQTT usando el API proporcionada por libMyQtt (la cual está compuesta por varias librerías: libmyqtt, libmyqtt-tls, libmyqtt-websocket). También se proporciona un servidor MQTT listo para usar llamado MyQttD que está construido usando libMyQtt. El servidor MyQttD es extensible añadiendo nuevos plugins escritos en C.

Existe una interfaz Python para libMyQtt llamada PyMyQtt de manera que es posible escribir servidores MQTT completos en Python.

  • Página oficial: http://www.aspl.es/myqtt

Fovap: una implementación de XSL-FO 1.0

Fovap es una implementación del estándar XSL-FO 1.0, que describe la manera de producir un formato de impresión con una descripción XML. Fovap está siendo usado para producir informes de alta calidad desde aplicaciones Af-Arch.

Af-Arch: un entorno para producir aplicaciones distribuidas

Af-Arch es un framework para desarrollar rápidamente aplicaciones distribuidas que almacenan sus datos en servidores SQL. Es una arquitectura orientada a servicios, con características de generación de código y con soporte para programar las interfaces cliente en C y C#.

  • Página oficial: http://www.aspl.es/fact

Shaper: una herramienta para obtener información XML de servidores Af-Arch

Shaper permite convertir cualquier servidor Af-Arch, y cualquier servidor BEEP con soporte XML-RPC, en una fuente de datos para producir documentos XML. Incluye una sintaxis embebida para realizar invocaciones de servicios, manipulando y almacenado los resultados obtenidos un documento XML. La herramienta tiene una sintaxis similar a XSLT.

  • Código fuente: https://dolphin.aspl.es/svn/publico/shaper/

Ju-Doc: Herramienta para generación de documentación de projectos Javascript

Ju-Doc es una herramienta que permite extraer la documentación del API de un proyecto inspeccionando la estructura del código Javascript y los comentarios que les acompañan. Ju-Doc utiliza el API Axl Stream para analizar el código fuente Javascript para producir una representación XML intermedia que luego es transformada en la salida final (HTML).

  • Código fuente: https://dolphin.aspl.es/svn/publico/ju-doc/
  • Ejemplo: http://www.aspl.es/jsVortex/docs

GMPC magnatune plugin

"Gnome music player client plugin" muestra una considerable reducción en su consumo de memoria después de cambiar de LibXML a Axl Library. Vea todos los detalles en la siguiente bitácora: http://blog.sarine.nl/2007/10/25/gmpc-magnatune-plugin-memory-reduction/

  • Página oficial: http://www.sarine.nl/gmpc

Cherokee/MonetDB Handler OSM

Una implementación de alto rendimiento de OSM usando Axl Library.

  • Página oficial: http://wiki.openstreetmap.org/index.php/Cherokee/MonetDB_Handler_OSM
